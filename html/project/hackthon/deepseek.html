Introduction to HTML

HTML (HyperText Markup Language) is the standard markup language for creating web pages. It describes the structure of a web page and consists of a series of elements that tell the browser how to display content.

Basic HTML Document Structure

Every HTML document follows a basic structure:

<!DOCTYPE html>
<html>
<head>
    <title>Page Title</title>
</head>
<body>
    <h1>My First Heading</h1>
    <p>My first paragraph.</p>
</body>
</html>

HTML Elements

HTML elements are the building blocks of HTML pages. Each element consists of a start tag, content, and an end tag:

<tagname>Content goes here...</tagname>

Remember: Some HTML elements (like <img>) are self-closing and don't have an end tag.

Next: Basic HTML Tags

Basic HTML Tags

HTML provides various tags to structure content on a web page. Here are some of the most commonly used ones:

Heading Tags

HTML has six levels of headings, from <h1> (most important) to <h6> (least important):

<h1>This is a Heading 1</h1>
<h2>This is a Heading 2</h2>
<h3>This is a Heading 3</h3>
<h4>This is a Heading 4</h4>
<h5>This is a Heading 5</h5>
<h6>This is a Heading 6</h6>

Paragraph and Text Formatting Tags

These tags help structure textual content:

<p>This is a paragraph.</p>
<strong>This text is important (bold)</strong>
<em>This text is emphasized (italic)</em>
<br>  <!-- Line break -->
<hr>  <!-- Horizontal rule -->

List Tags

HTML supports ordered (numbered) and unordered (bulleted) lists:

<ul>
    <li>Unordered list item 1</li>
    <li>Unordered list item 2</li>
</ul>

<ol>
    <li>Ordered list item 1</li>
    <li>Ordered list item 2</li>
</ol>
Next: Links and Images

Links and Images

Links and images are essential elements of most web pages. HTML provides simple ways to add both.

Anchor Tags (Links)

The <a> tag defines a hyperlink, which is used to link from one page to another:

<a href="https://www.example.com">Visit Example.com</a>
<a href="about.html">About Us</a>
<a href="#section1">Jump to Section 1</a>  <!-- Internal link -->

Image Tags

The <img> tag is used to embed an image in an HTML page. It is an empty tag containing only attributes:

<img src="image.jpg" alt="Description of image">
<img src="photo.png" alt="A beautiful photo" width="500" height="600">

Important: Always use the alt attribute to provide alternative text for images. This is crucial for accessibility and SEO.

Combining Links and Images

You can use an image as a link by nesting an <img> tag inside an <a> tag:

<a href="https://www.example.com">
    <img src="logo.png" alt="Example Logo">
</a>
Next: HTML Tables

HTML Tables

HTML tables allow you to arrange data into rows and columns. They are created using the <table> element.

Basic Table Structure

A basic table consists of the <table> element, one or more <tr> elements (table rows), and <td> elements (table data cells):

<table>
    <tr>
        <th>Name</th>
        <th>Age</th>
        <th>Country</th>
    </tr>
    <tr>
        <td>John</td>
        <td>25</td>
        <td>USA</td>
    </tr>
    <tr>
        <td>Maria</td>
        <td>32</td>
        <td>Spain</td>
    </tr>
</table>

Table Headers

Use the <th> tag to define header cells. Most browsers display <th> elements in bold and centered:

<table>
    <tr>
        <th>Product</th>
        <th>Price</th>
        <th>Quantity</th>
    </tr>
    <tr>
        <td>Apple</td>
        <td>$1.00</td>
        <td>10</td>
    </tr>
</table>

Table Styling Attributes

While CSS is preferred for styling, some attributes can be used directly in HTML:

<table border="1" cellpadding="10" cellspacing="0" width="100%">
    <tr>
        <th>Header 1</th>
        <th>Header 2</th>
    </tr>
    <tr>
        <td>Data 1</td>
        <td>Data 2</td>
    </tr>
</table>

Note: While these attributes work, it's generally recommended to use CSS for styling tables in modern web development.

Next: HTML Forms

HTML Forms

HTML forms are used to collect user input. A form contains input elements like text fields, checkboxes, radio buttons, submit buttons, and more.

Basic Form Structure

The <form> element defines a form that can contain various input elements:

<form action="/submit-form" method="post">
    <label for="fname">First name:</label><br>
    <input type="text" id="fname" name="fname"><br>
    
    <label for="lname">Last name:</label><br>
    <input type="text" id="lname" name="lname"><br>
    
    <input type="submit" value="Submit">
</form>

Common Input Types

HTML provides various input types for different kinds of data:

<input type="text">      <!-- Text input -->
<input type="password"> <!-- Password field -->
<input type="email">    <!-- Email address -->
<input type="number">   <!-- Numeric input -->
<input type="date">     <!-- Date picker -->
<input type="checkbox"> <!-- Checkbox -->
<input type="radio">    <!-- Radio button -->
<input type="file">     <!-- File upload -->
<input type="submit">   <!-- Submit button -->

Select Dropdowns and Textareas

For longer text inputs or multiple options, use these elements:

<select name="cars">
    <option value="volvo">Volvo</option>
    <option value="saab">Saab</option>
    <option value="mercedes">Mercedes</option>
</select>

<textarea name="message" rows="10" cols="30">
Default text goes here...
</textarea>
Next: Semantic HTML

Semantic HTML

Semantic HTML introduces meaning to the web page rather than just presentation. It makes web content more accessible and helps with SEO.

What are Semantic Elements?

Semantic elements clearly describe their meaning to both the browser and developer. Examples of non-semantic elements: <div> and <span> - they tell nothing about their content. Examples of semantic elements: <form>, <table>, and <article> - they clearly define their content.

Common Semantic Tags

HTML5 introduced many new semantic elements:

<header>Defines a header for a document or section</header>
<nav>Defines navigation links</nav>
<main>Specifies the main content of a document</main>
<section>Defines a section in a document</section>
<article>Defines independent, self-contained content</article>
<aside>Defines content aside from the page content</aside>
<footer>Defines a footer for a document or section</footer>
<figure>Specifies self-contained content</figure>
<figcaption>Defines a caption for a <figure> element</figcaption>
<mark>Defines marked/highlighted text</mark>
<time>Defines a date/time</time>

Example Page Layout with Semantic Elements

<body>
    <header>
        <h1>Website Title</h1>
        <nav>
            <ul>
                <li><a href="#">Home</a></li>
                <li><a href="#">About</a></li>
            </ul>
        </nav>
    </header>
    
    <main>
        <article>
            <h2>Article Title</h2>
            <p>Article content...</p>
        </article>
        
        <aside>
            <h3>Related Links</h3>
            <ul>
                <li><a href="#">Link 1</a></li>
            </ul>
        </aside>
    </main>
    
    <footer>
        <p>Copyright Info</p>
    </footer>
</body>

Benefits: Using semantic HTML improves accessibility, helps with SEO, makes code easier to read and maintain, and enhances interoperability.
